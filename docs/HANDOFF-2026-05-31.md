## The call-number ordering rule (owner-confirmed — durable; don't regress)

Used by overlap detection, sorting, and editor range access-control. See issue #100.

- **Default:** plain **string comparison** of the raw value. Works because Dewey integer
  parts are zero-padded to 3 digits (magnitude correct); `'(' < '.'` makes a parenthetical
  sub-classification sort right after the base and before decimals (`396(44) < 396.04 < 396.4`);
  leading zeros are significant (`320(044) < 320(44)`); different-length parentheticals order
  digit-by-digit (`323.67(6761) < 323.67(73)`).
- **Exception — `ML` / `MT` prefixes:** compare the prefix, then the number after it as a
  **natural number** (`ML5 < ML113`). (`E…` / single-`M…` left on default string compare —
  revisit only if needed.)
- **Overlap:** `start1 < end2 && start2 < end1` (strict) + an identical-ranges guard. Touching
  boundaries and identical ranges are NOT conflicts; a point range inside a wider one IS.
- **Implementation (kept behaviorally identical / parity-tested):** `compareCallNumbers` in
  `admin/services/data-model.js`, `admin/utils/range-filter.js`, `lambda/range-validation.mjs`.
  `parseRangeValue`/`parseCallNumber`'s float is retired from the ordering path (kept only for
  prefix/format checks).

## Lambda follow-up (optional — does NOT affect the dashboard count)

`range-validation.mjs` is bundled into 5 functions: `putCsv`, `updateUser`, `validateStaging`,
`promoteStaging`, `applyReconcileToStaging`. There is **no build script**; the safe method is a
surgical single-file swap per function (so nothing else changes vs what's live):

```bash
# (after AWS login) for each fn:
url=$(aws lambda get-function --function-name "primo-maps-$fn" --query Code.Location --output text)
curl -s "$url" -o /tmp/$fn.zip
( cd /tmp && cp /path/to/repo/lambda/range-validation.mjs . && zip /tmp/$fn.zip range-validation.mjs )
aws lambda update-function-code --function-name "primo-maps-$fn" --zip-file fileb:///tmp/$fn.zip
```
Only matters for the server-side "editor may only edit rows in their assigned range" check, and
only when a boundary is a parenthetical call number (rare). Do it on/after this work, no rush.

## Gotchas observed this session

- **AWS SSO tokens expire** within a session — re-auth before any `aws`/`redeploy.sh`.
- **`?v=` is inconsistent** across admin module imports (#27) — `redeploy.sh` relies on CloudFront
  invalidation + hard-refresh, not `?v=` bumps.
- **jest `unstable_mockModule` mocks go stale** — adding a new export to `data-model.js` broke
  two csv-editor tests whose partial mock omitted it; keep those mocks in sync.
- **`pkill -f "http-server"`** matches its own command and kills the shell — use the
  `[h]ttp-server` bracket trick.
- **`redeploy.sh` syncs `admin/` only** (with `--delete`) — it does not touch the `mockups/`
  prefix, so deploying admin won't disturb the mockup.
